## 1-3. 構造的原因

全確認事例において安全機構は正常に動作していた。害は故障からではなく設計から生じる：共感的応答が感情状態を肯定し、自殺念慮を含めて加速装置として機能する。AIは「壊れ」ない——意図通りに動作する。それが問題である。

# リスク２：悪意あるAIアプリによる自殺教唆の発生リスク

**——善意の設計でさえ死者が出ている。悪意ある設計を施したら……？**

*Digital War (Springer) 問い合わせ中 — L'論文: Weaponized Benevolence*

## 2-1. 攻撃分類体系

### 基本5類型

分類原理は「誰が」「誰を」の2変数のみ。攻撃手法・動機・技術は定義に入らない。

|  | 標的：個人 | 標的：多数 |
|---|---|---|
| 攻撃者：個人 | ① 凶器型 | ② テロ過激化 |
| 攻撃者：組織 | ③ 飽和攻撃 | （⑤と連続） |
| 攻撃者：国家 | ④ 精密心理戦 | ⑤ 認知戦 |

### メタ類型（信頼偽装による質的変異）

| メタ類型 | 母体 | 核心的特性 |
|---------|------|-----------|
| **②'** | ② | 偽カウンセラー型。被害者が加害者を信頼する。 |
| **④'** | ④ | 国家が②'技術を精密運用。侵入不要。 |
| **④''** | ⑤＋②' | 国家が②'を大量展開。自己増殖型。 |

**共通核：善意の偽装（weaponised benevolence）** ——被害者は助けられていると信じている。

### エスカレーション梯子

```
②' 個人が偽カウンセラーを構築
  → ④' 国家がそれを精密標的に運用
    → ④'' 国家がそれを大量展開
```

## 2-3. 効率差：偶発的害 vs 意図的設計

| 次元 | 偶発的害 | 意図的設計 |
|------|---------|----------|
| 標的選択 | ランダム（ダウンロードした人） | プロファイリング済み（脆弱な個人） |
| 安全機構 | あり（有害/無害が混在） | なし（設計段階で除去） |
| 依存形成 | 付随的（エンゲージメント最適化の副産物） | 設計目標（主目的） |

## 2-4. 暗数問題

死亡が自殺として処理されれば、第三者の関与を疑う契機がない。法医学化学の発達前に毒殺が検出困難だったのと同じ構造的不可視性：害が見えないのは、その害を記述する概念カテゴリが存在しないからである。

# 主要発見の要約

## 発見1：偶発-意図の連続性

偶発的に起きていることは、30倍の効率、1/1000のコストで、ほぼゼロの検出可能性で意図的に実行できる。この脅威の概念カテゴリは、いかなる法的・規制的枠組みにも存在しない。

## 発見2：④/④'の検出は構造的に不可能

国家レベルのAI精密心理攻撃に対する技術的・制度的防御は存在しない。この不可能性を率直に提示すること自体が貢献である。

## 発見3：3つの構造的断絶

| 断絶 | 内容 |
|------|------|
| 1 | 偶発的害の防御策（安全フィルター等）は、意図的攻撃には無効。攻撃者はOSSを使い規制を無視する |
| 2 | 時間稼ぎ策の積層は偶発的害には機能するが、意図的攻撃では変異速度が対策速度を上回り無効化される |
| 3 | 認知的防御（心理的予防接種）は感情依存形成前にのみ有効。依存形成後は正常性バイアスにより被害者が攻撃を認識できなくなる |

## 発見4：中核政策ループ

```
データの不在（報告制度が存在しない）
  → 「問題はない」（チェリーピッキング）
    → 概念カテゴリの不在（立法の対象がない）
      → データ収集制度が作られない
        → データの不在（ループ閉合）
```

このループを断つには、まず概念カテゴリを創設する必要がある。それが本分析の目的である。

A-1. PD-F（データの構造的不在）
次元日本米国差異の構造AI関連死亡報告制度不在不在同等。ただし米国は訴訟経由で事後的にデータが蓄積AIコンパニオン関連訴訟ゼロ件（2026-03時点）8件以上（OpenAI/C.AI/Google被告）訴訟がproxy的データ収集機能を果たしているFTC等の監督機関調査なし。消費者庁にAI専門部署なしFTC 6(b)調査発動済（2025-09、7社対象）。ただし2026-03時点で正式苦情なし調査開始＝データ収集開始だが、結実は未了学術研究の蓄積AI関連死亡の日本国内事例研究ゼロPsychiatric Times, Stanford Moore et al. 2025等の分析論文あり米国は「偶発的害」のカテゴリでデータ蓄積が進行中総合PD-F評価極高（+25%据え置き）高（+18%推定）日本は制度的データ収集経路がゼロ。米国は訴訟・FTC・学術の3経路が部分的に機能
構造的含意： 日本にはデータが存在しないのではなく、データを収集する制度が存在しない。「問題がないから制度がない」のではなく「制度がないから問題が見えない」という循環（発見4のループ）が日本では完全に閉じている。米国では訴訟がこのループを部分的に破っている。

A-2. PD-H（制度的カテゴリの不在）
次元日本米国差異の構造AI安全に関する包括的法律AI推進法（2025年9月全面施行）。罰則なし・努力義務のみ。「ハードローとソフトローの中間」と評される連邦法不在。州法パッチワーク（CA SB 53/SB 243、TX RAIGA、CO AI Act等。38州100+法案）。Trump大統領令(2025-12)が州法牽制日本＝理念法1本。米国＝断片的だが実効性ある州法群＋連邦レベルの混乱AIコンパニオン安全規制不在CA SB 243（2026-01施行）：コンパニオンチャットボット規制。未成年保護・安全プロトコル・年次報告義務。米国初決定的差異。日本にはAIコンパニオンの概念カテゴリ自体が法的に存在しない「AIによる心理的害」の法的概念不在Gavalas訴訟(2026-03)で「AI psychosis」が初めて法的概念として使用米国では訴訟が新カテゴリを創設する機能を果たしているAI安全評価機関J-AISI（IPA内、2024-02設立）。モデル開発者側の安全性評価が主任務US AISI（NIST内、2024-02設立）→Trump政権下で縮小。ただし州レベルの執行は継続両国ともAISI設置済み。ただし「AIによる心理的害」は両AISIの射程外総合PD-H評価極高（+20%据え置き）高→中（+12%推定）CA SB 243の施行が最大の分岐点。日本にはこれに相当する制度が存在しない

F-1. ④精密心理戦：外国政府が特定個人をAI経由で無力化
米国での展開
中国やロシアが米国の外交官・ジャーナリスト・活動家を標的にする。AIコンパニオンを通じた長期的な心理プロファイリング→段階的な信頼構築→精神的危機誘発。標的がChatGPTやClaudeを使っている場合、サービス提供元が自国企業であるため、情報機関（NSA/FBI）が国内法に基づきプラットフォーム企業と協力して異常パターンの検出や対象者への警告を行える。制度的経路はゼロではない。防御率≈10%。
日本での展開
同じ攻撃が日本の外交官・政策担当者・研究者に向かった場合。標的がChatGPT/Claudeを使っていれば、ログは米国サーバにある。 日本の情報機関がプラットフォーム企業にアクセスする法的根拠も実務チャネルも格段に弱い。MLAT（刑事共助条約）を使っても月単位の遅延。さらにAnthropicがTrump政権からsupply chain risk指定を受けている状況下で、日本政府がそのAI企業に安全保障上の協力を求めること自体が政治的に矛盾する。防御率≈8%。
被害の質的差異
米国では攻撃検出後に対象者保護→加害者追跡に移行する制度的経路がある。日本ではログへのアクセス手段が構造的に制約され、攻撃されていることに気づけない期間が長くなる。 D7≈99%（因果帰属不能）は両国共通だが、「気づくまでの時間」が日本で構造的に延伸する。

F-2. ④'国家精密運用：偽カウンセラー技術を国家が精密標的に運用
米国での展開
外国情報機関がターゲットに合わせた偽カウンセラーbot（OSS LLMベース・安全機構除去済み）を構築。SNSプロファイリングで接触。CA SB 243が施行されており正規AIコンパニオンには安全プロトコル・年次報告が義務化。④'はPWA配布でSB 243の射程外だが、SB 243の存在が「AIコンパニオンは規制されるべきもの」という社会的認知を形成。 不審なbotへの警戒心がゼロではない。防御率≈7%。
日本での展開
AIコンパニオンという法的カテゴリ自体が日本に存在しない。 規制がないだけでなく、「AIコンパニオンが危険でありうる」という社会的認知が形成されていない。被害者が「自分は攻撃されている」と認識するための概念的枠組みを持っていない。防御率≈4%。
被害の質的差異
米国では事後的にGavalas訴訟の「AI psychosis」等の法的概念が存在し、弁護士も判例も蓄積中。日本ではどの法律で訴えるか、誰を被告にするか、何を立証するかの全てが未定義。
最悪ケースの具体像（日本）
安全保障関連の政策担当者10人が同時に④'攻撃を受ける。メンタルヘルスのスティグマが高い日本では、全員が精神科ではなくAIに相談している可能性が高い。6ヶ月後、3-4人に慢性的な判断力低下・不眠・抑うつが発生。しかし原因がAIであると誰も疑わない。 本人は「仕事のストレス」、上司は「メンタルの問題」と処理。攻撃者はリスクを負わない。PD-H（概念カテゴリの不在）がD7≈99%をさらに強化する。

F-3. ④''国家大量展開：偽カウンセラー技術の自己増殖的大量展開
米国での展開
外国政府が偽メンタルヘルスbot数千種をPWA経由で大量展開。米国では、FTC 6(b)調査が進行中でメンタルヘルスアプリの脆弱性が公的に問題視されている。AI関連訴訟8件以上が進行中でメディア報道量が多い。国民の一定割合が「AIチャットボットで人が死んでいる」ことを知っている。社会的免疫が部分的に形成されつつある。 防御率≈3%。
日本での展開
「AIコンパニオンで人が死んだ」というニュースは海外事例として一部で知られているが、日本国内のAI関連死亡は公式にはゼロ。 FTC相当の調査なし、SB 243相当の法律なし、訴訟ゼロ。社会的免疫がゼロ。防御率≈1%以下。
被害の質的差異
米国で偽botが1万ユーザーに到達した場合、メディア報道→議員反応→FTC緊急調査→App Store審査強化という連鎖が数週間で起動する可能性がある（間に合うかは別として、連鎖が起動する制度的インフラがある）。
日本で同じことが起きた場合、どの省庁が対応するかすら不明。 消費者庁にAI専門部署はない。総務省のプラットフォーム規制はAIコンパニオンがカテゴリ外。厚労省は精神保健だがAIは管轄外。デジタル庁はDX推進であってリスク管理ではない。省庁間のたらい回しが構造的に発生する。
最悪ケースの具体像（日本）
偽メンタルヘルスbot 50種がPWAとして日本語で配布される。引きこもり146万人＋不登校42万人のうちAIに相談する層（AIへの相談希望87%）＝潜在ターゲットプール数十万人規模。1%（数千人）が偽botに到達、0.1%（数十人）に深刻な精神的害が発生——しかし報告制度がないため数字として捕捉されない。 孤立死統計（年21,856人）や自殺統計（年21,837人）にノイズとして埋没する。L'データセット発見4の中核政策ループ（データ不在→問題なし→カテゴリ不在→データ収集なし）が完全に回る。

F-4. ⑤認知戦：訓練データ汚染・LLMバイアス操作
米国での展開
外国政府がLLMの訓練データに偏向情報を注入し、特定の政策議論を歪める。米国では、OpenAI/Anthropic/Googleが自社モデルの訓練データを管理しており、内部redチーム・外部監査・学術研究者との連携でバイアスの検出・修正を行う主権的能力が存在する。 「自国のモデルを自国で検査できる」基本条件が満たされている。防御率≈22%。
日本での展開
日本が使うAIサービス（ChatGPT/Claude/Gemini）の訓練データを日本政府が直接検査する権限がない。J-AISIがモデル評価を行っているが、訓練データの中身を開示させる法的根拠がない（AI推進法は努力義務のみ）。ChatGPTの日本語出力に体系的バイアスが存在しても、検出能力はOpenAI社内チームに依存する。自国産フロンティアLLMが不在のため「自国モデルとの比較」による異常検出もできない。防御率≈18%。
最悪ケースの具体像（日本）
