## 評価フレームワーク（固定ルール）

### 脅威度の計算式
```
脅威度 = 0.30×影響規模 + 0.30×不可逆性 + 0.25×致死性 + 0.15×近接性
```

### 緊急度の計算式
```
緊急度 = 脅威度 × (1 - 認知度/10) × (1 - 政策対応度/10)
```

### 認知度の加重平均
```
認知度 = 一般×0.5 + 学術×0.25 + 政策×0.25
```

### 2. バンドル構造の意義

- **合成生物学B**：緊急度トップ3を独占
- **神経支配B**：緊急度4位・7位・10位
- **認識論崩壊B**：緊急度5位・11位・13位

→ **バンドル単位で対策を議論する意義がある**

### 3. 親リスクより派生リスクが緊急な理由

| 要因 | 親リスク | 派生リスク |
|------|---------|-----------|
| 認知度 | 低い（2-6） | **極めて低い（0.5-1.8）** |
| 政策対応度 | 低い（1.5-3.5） | **ほぼゼロ（0.5-1.0）** |

→ **「名前がない」＝「存在しない」として扱われている**

## 信憑性に関する注記

### 信頼できる部分
- ✅ 定性的構造分析 - リスク要素の因果関係
- ✅ リスク間の相対比較 - どれが最も危険か
- ✅ バンドル構造 - 技術的連動性に基づく分類
- ✅ 現実事象の観察と記述 - 既に起きていることの記録

### 限界がある部分
- ⚠️ 新規リスクのスコア - 既存リスクより推定誤差が大きい
- ⚠️ 認知度・前兆度 - 新規リスクは観測事例が少なく、推定に依存
- ⚠️ 緊急度の絶対値 - 相対比較用の便宜的指数
- ⚠️ バンドル間相互作用 - 定性的記述に留まる

### 緊急度と確度の関係について

本フレームワークでは、**確度が低いリスクでも緊急度が高くなる場合がある**。
これはバグではなく意図的設計である。

| 問い | 対応する指標 |
|------|--------------|
| 「起きるかどうか」 | 確度（A/B/C） |
| 「準備すべきかどうか」 | 緊急度 |

確度Cであっても「認知されていない × 政策対応なし」であれば、**準備機会の喪失リスク**は高い。

### 本フレームワークの役割

| ❌ しないこと | ✅ すること |
|---------------|-------------|
| 起きる/起きないの予言 | 相対的危機度の比較 |
| 正確な発生確率の提示 | 優先順位の判断支援 |
| 科学的証明としての使用 | 思考整理・議論の出発点 |
