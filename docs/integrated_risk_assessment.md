## 評価フレームワーク（固定ルール）

### 脅威度の計算式
```
脅威度 = 0.30×影響規模 + 0.30×不可逆性 + 0.25×致死性 + 0.15×近接性
```

### 緊急度の計算式
```
緊急度 = 脅威度 × (1 - 認知度/10) × (1 - 政策対応度/10)
```


## メタリスク枠（フレームワーク外）

> **このリスク表自体が無効になる条件を示す。コアリスクとの同列比較は不適切。**

### ASI（人工超知能）出現

| 項目 | 内容 |
|------|------|
| 定義 | AGI→ASIへの移行により、人類の認知・制御能力を根本的に超越した知性が出現するリスク |
| 臨界点 | 2030-2100年（専門家間で大分岐） |
| 確度 | C（低）- 時期・形態とも不確実性が極めて高い |

#### 評価スコア（参考値）

| 要素 | スコア | 根拠 |
|------|--------|------|
| 影響規模 | 100 | 定義上、全人類・全システムに影響 |
| 不可逆性 | 100 | 定義上、完全不可逆 |
| 致死性 | 70 | シナリオ依存（友好的50↔敵対的95）中央値 |
| 近接性 | 55 | 専門家間で大分岐、中央値採用 |
| **脅威度** | **95.75** 🔴 | 全リスク中最高 |

| 指標 | スコア | 根拠 |
|------|--------|------|
| 認知度 | 7.0 | AI安全性研究活発、SF経由の一般認知も高い |
| 前兆度 | 5.5 | LLM急進展、ただしAGI未到達 |
| 政策対応度 | 3.0 | EU AI Act等あり、ASI特化は皆無 |
| **緊急度** | **46** | 高認知が緊急度を抑制（参考値） |

#### なぜメタリスク枠なのか

| コアリスクとの違い | 説明 |
|-------------------|------|
| 評価軸の破壊 | 「脅威度」「認知度」等の概念自体が無意味化 |
| 全リスクの上書き | 他リスクすべてがASIの意図に従属 |
| 人類の主体性喪失 | 「準備する」という行為の前提が崩れる |
| 予測の原理的限界 | 人類より賢い存在の行動を人類が予測できない |

## 発見事項

### パターン1：高脅威×高認知 → 緊急度低（参照リスク）

| リスク | 脅威度 | 認知度 | 緊急度 |
|--------|--------|--------|--------|
| 核戦争 | 86.00 | 7.5 | 22 |
| 気候変動ティッピング | 83.00 | 8.0 | 17 |

→ **「すでに議論されている」ことが緊急度を下げる**
→ 本プロジェクトの「低認知フォーカス」設計意図と整合

### パターン2：高脅威×低認知 → 緊急度高（コアリスク）

| リスク | 脅威度 | 認知度 | 緊急度 |
|--------|--------|--------|--------|
| バイオテロ民主化 | 86.25 | 0.8 | 76 |
| 生態系ハイジャック | 80.25 | 1.0 | 71 |

→ **「名前がない＝存在しない」問題**
→ 準備機会の喪失リスクが最大化

### パターン3：メタリスク（ASI）の特殊性

| リスク | 脅威度 | 認知度 | 緊急度 |
|--------|--------|--------|--------|
| ASI出現 | 95.75 | 7.0 | 46 |

→ **脅威度最高だが、緊急度は中位**
→ 高認知が緊急度を抑制
→ ただし「フレームワーク破壊」の性質上、同列比較は不適切

### パターン4：構造的位置づけの意義（新規）

| 位置 | リスク例 | 特性 |
|------|---------|------|
| 基盤 | マイクロプラ | 横断的浸透、他リスクの土台 |
| バンドル | 3親+6派生 | 技術因果による親子構造 |
| ハブ | A/P紛争 | 複数バンドルの結節点 |
| 衛星 | 量子系、遺伝子格差 | ハブ周辺の技術特化リスク |

→ **「独立リスク」は実は独立していない**
→ 構造図により相互関係が可視化

## 信憑性に関する注記

> **本リスク表は、将来リスクの相対比較と議論整理を目的としたものであり、
> 発生確率や時期を確定するものではない。
> 確度Cリスクは推測的シナリオとして明示的に扱う。**

### 信頼できる部分
- ✅ 定性的構造分析 - リスク要素の因果関係
- ✅ リスク間の相対比較 - どれが最も危険か
- ✅ 4層構造の設計 - 「なぜこれを扱い、なぜあれを扱わないか」の説明
- ✅ コアリスク14の構造図 - 基盤→バンドル→ハブの関係性

### 限界がある部分
- ⚠️ 新規リスクのスコア - 既存リスクより推定誤差が大きい
- ⚠️ 認知度・前兆度 - 新規リスクは観測事例が少なく、推定に依存
- ⚠️ 緊急度の絶対値 - 相対比較用の便宜的指数

### 緊急度と確度の関係について

本フレームワークでは、**確度が低いリスクでも緊急度が高くなる場合がある**。
これはバグではなく意図的設計である。

| 問い | 対応する指標 |
|------|--------------|
| 「起きるかどうか」 | 確度（A/B/C） |
| 「準備すべきかどうか」 | 緊急度 |

確度Cであっても「認知されていない × 政策対応なし」であれば、**準備機会の喪失リスク**は高い。

### 4層構造の役割

| 層 | 役割 | 本プロジェクトでの扱い |
|----|------|----------------------|
| メタリスク | フレームワーク無効化条件の監視 | 別枠で明示 |
| コアリスク | 低認知×高脅威の可視化 | **主要対象** |
| 補完リスク | 視野の横拡張 | 保険的記載 |
| 参照リスク | 比較軸の固定 | 役割外だが基準点 |

### 本フレームワークの役割

| ❌ しないこと | ✅ すること |
|---------------|-------------|
| 起きる/起きないの予言 | 相対的危機度の比較 |
| 正確な発生確率の提示 | 優先順位の判断支援 |
| 科学的証明としての使用 | 思考整理・議論の出発点 |

**本表は「未来を当てに行く道具」ではなく「当てに行きたくなる衝動を抑える道具」として設計されている。**
