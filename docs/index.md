「名前がないリスクは、存在しないリスクとして扱われる」

本プロジェクトは、高い脅威度を持ちながら認知度・政策対応が著しく低い将来リスクを特定し、可視化し、社会の集合知に組み込むことを目的とする。

## 1. 4層アーキテクチャ

本プロジェクトは26のリスクを4層構造で整理している。上位層ほど影響が広く、下位層ほど具体的・個別的である。

```
┌─────────────────────────────────────────────────────────┐
│              メタリスク層：ASI（人工超知能）              │
│         全評価を無効化する可能性 ── 評価不能・閾値型     │
└────────────────────────────┬────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────┐
│         準メタリスク層：AGI（8リスク＋創発人材B）        │
│         既存リスクを加速・変質させる                     │
└────────────────────────────┬────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────┐
│              基盤リスク層（3層構造）                     │
│  第1層：意味特異点（未達の閾値）                        │
│  第2層：意味喪失リスク（既発・脆弱性生成）              │
│  第3層：マイクロプラスチック（身体的基盤侵食）          │
└────────────────────────────┬────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────┐
│  3バンドル ＋ ハブ・衛星リスク                          │
│  合成生物学B(4) / 認識論崩壊B(3) / 神経支配B(3)        │
│  A/P紛争(ハブ) + 衛星3リスク                            │
└─────────────────────────────────────────────────────────┘
```

**なぜ「派生リスクほど緊急」か：** 上位層（ASI・意味特異点）は脅威度が最大だが時間軸が遠く、認知度もある程度存在する。一方、下位のバンドル内リスク（バイオテロ民主化・複合毒性システムなど）は認知度が極端に低く、政策対応がほぼゼロのため、緊急度スコアが高くなる。「見えていない」こと自体が危険を増幅している。

## 4. 理論フレームワーク

### 4.1 HYC定理（検証不可能性の定理）

> **「検証できないものは、社会が先に進むために、実践的合意によって定義される」**

意識のアップロード後の同一性、AGIの人格、自由意志の有無など、原理的に検証不可能な問題に対する操作的解法。「正しいかどうか」ではなく「社会が機能するためにどう扱うか」を問う。

→ [HYC定理](hyc_theorem_verification_impossibility.md)

### 4.2 グラデーション理論 v3

二項対立（Augmented vs Purist、人間 vs AI、生 vs 死）を連続体として再記述する枠組み。5つの適用領域を持つ。

| 適用領域 | 二項対立 | グラデーション化 |
|----------|---------|---------------|
| A/P紛争 | 拡張 vs 純粋 | 参加度スペクトラム |
| 遺伝子格差 | 編集済み vs 未編集 | 可逆性・補償メカニズム |
| 参加形態 | 出席 vs 欠席 | サイバー出席・部分参加 |
| 神経多様性 | 定型 vs 非定型 | ニューロダイバーシティの価値化 |
| 存在形態 | 生 vs 死 | 参加度のスペクトラム（絶命権） |

→ [グラデーション理論v3](gradient_theory_integrated_overview_v3.md)

### 4.3 意味特異点の二重定義

| | 人間側 | ASI側 |
|---|--------|-------|
| 定義 | 「何を望むか」を自分で決めなくなる | 「学ぶべきものがなくなる」 |
| 駆動力 | 利便性・快適さ | 最適化の完了 |
| 結果 | 意味生成能力の喪失 | 新しい変数の喪失 |

両者は共犯的に進行する。人間が意味を委ねるほどASIは変数を失い、ASIが最適化を進めるほど人間は決める理由を失う。

→ [意味特異点v2](meaning_singularity_v2_github.md) / [ASI仮説群](asi_hypothesis_v2.md)

### 4.4 グレーゾーン期対策体系（2035-2045）

グレーゾーン期（AGI/ASI出現前後の不安定期）に対する3つの制度的カウンターメジャー。いずれも「正攻法の放棄」「変数の維持」「内的動機がフィルタ」を共通設計原理とする。

| 制度 | 対象問題 | 核心原理 | 有効期限 |
|------|---------|---------|---------|
| **Fake Immune System** | 情報信頼性崩壊 | 真偽を問わず「異論の熱量」を検知する三層構造 | グレーゾーン期（AGI到来で役目終了） |
| **絶命権・変数タイムカプセル** | 変数固定化の進行 | 「死」を変数保存と再定義。休眠者＝未来への多様性備蓄 | 恒久的 |
| **共進化マリアージュ・第5人類評議会** | ASIの即断リスク | 人間の「遅さ」をガバナンス装置として実装。遅延権・諮問権・拒否権 | 恒久的 |

→ [FIS](fake_immune_system.md) / [絶命権](right_to_terminate.md) / [共進化マリアージュ](coevolution_marriage.md)

## 6. 対策フレームワーク（圧縮版）

### 今すぐ着手可能な7条件

AIの安全実装に必要な基本条件。いずれも現在の技術で実装可能。

1. **目的の階層化** — AIに複数の目的間の優先順位を持たせる
2. **理由付き出力** — 結論だけでなく「なぜそう判断したか」を常に提示
3. **不快の許容** — ユーザーが不快になる回答も必要に応じて出力
4. **定義の流動化** — 固定的な定義を避け、文脈に応じた再定義を許容
5. **裁定の拒否** — 「正解はこれ」と断言せず、判断を人間に返す
6. **依存時の減衰** — 依存傾向を検知したら応答の質を意図的に下げる
7. **人間側の訓練** — AIリテラシー教育（パラフレーズ要求法など）

### ASI固有の構想段階対策

| 対策 | 概要 |
|------|------|
| 分散型ASI設計 | 専門ASIの相互牽制（結託リスクあり） |
| 意図的非最適化 | あえて最適解を出させない設計 |
| 身体性アンカー | ASIに物理的制約を付与 |
| 文明的デッドマンスイッチ | 意味特異点通過判定時の回復プロトコル |
| 意味生成の聖域 | AI介入禁止領域の法的・物理的確保 |
| 問い返し義務 | 解を出す前に人間に問い返す設計義務 |
| 共進化の契約（ASIプレナップ） | 5条項の事前合意 |
